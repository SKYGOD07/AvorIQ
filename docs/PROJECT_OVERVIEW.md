## 1. What is AvorIQ?

**AvorIQ** (Adaptive Vision for Opportunity and Resources Intelligence Quotient) is an AI-powered education companion built for Indian students from **Class 6 to Post-Graduation**. The project's V1 (currently active) is a **Scholarship Intelligence Platform** that aggregates, structures, and matches scholarships against a student's socioeconomic and academic profile.

### Mission
Democratise access to financial support for students in rural villages, government schools, and tier-2/3 cities who miss opportunities due to fragmented information, complex eligibility rules, and language barriers.

### Core features (V1)
- **Smart Match Finder** — profile-driven eligibility scoring across scholarships.
- **Advanced Filters** — by category (Govt / Private / NGO / International), education level, amount, deadline.
- **AI Chat Assistant** — Perplexity-style conversational search with optional local LLM backend.
- **Application Tracker** — milestone pipeline (Saved → Checked → Applied → Approved → Funds).
- **Detailed Modals** — eligibility, documents, application steps, FAQs.
- **Save / Bookmark** — shortlist from a personal dashboard.
- **Auth + Profile Questionnaire** — Firebase email / Google sign-in plus a 3-step profile builder.
- **Brutalist UI** — Neo-brutalist design system (Bauhaus palette: charcoal + red + yellow).

### Roadmap modules (planned)
V2 — AI Study Planner, Exam Prep Assistant, WhatsApp AI Agent, Mental Wellness.
V3 — RAG Vector Search Engine, Career Navigator, YouTube Companion.
V4 — Community Forums, Parent Dashboard.

## 3. How the system works

### Frontend (Next.js 16 App Router, React 19, TypeScript)
- Pages: `/`, `/login`, `/questionnaire`, `/scholarships`, `/chat`, `/saved`, `/coming-soon`, `/about`.
- Routing/auth gating: `context/AuthContext.tsx` + `components/RouteGuard.tsx`.
- Auth: Firebase (email + Google). Falls back to a **mock auth** when env vars are unset.
- State: persisted via `useLocalStorage` for chat history, saved scholarships, progress milestones, and guest message counter.
- Chat: `components/ChatEngine.tsx` streams from `/api/chat` (proxy → FastAPI) when the backend is reachable; otherwise runs the local keyword-based parser (`parseAndMatch`).
- Data source: `data/scholarships.ts` (a curated set of realistic Indian scholarships with eligibility, documents, FAQs). Match logic in `utils/matcher.ts`.
- Styling: Tailwind CSS v4 + custom `@theme` palette defined in `app/globals.css` (Bauhaus brutalist look).
- Animation: Framer Motion, Canvas Confetti, ReactBits components.

### Backend (FastAPI, Python)
- Started via `docker compose up -d postgres ollama backend` (see `scripts/setup.ps1`).
- Endpoints:
  - `GET /health`
  - `POST /chat/event` — record a user/AI message and update profile
  - `POST /profiles/upsert` — merge extracted profile signals
  - `POST /collect` — kick off a `collect_for_profile` job (runs scraper, scores results, persists them)
  - `GET /profiles/{user_id}`
  - `GET /jobs/{job_id}`
- Storage: SQLite via `storage.py` (single file `avoriq.db`). Postgres+pgvector service is declared in compose for future use.
- LLM: Ollama container (`gemma3:4b` chat, `nomic-embed-text` embeddings) — pulled by `setup.ps1`.
- Scraping: `scripts/scrape_scholarships.py` produces `scholarships.csv`, `raw_records.jsonl`, `normalized_scholarships.csv`, `gemma_instruction_dataset.jsonl`, `summary.json` in `datasets/scholarships/`.
- Trust scoring: `authenticity.py` (rules for `.gov.in`, `.nic.in`, `.ac.in`, source heuristics) → labels `authentic` / `review` / `low_trust`.
- Relevance scoring: `profile_memory.py` (regex profile extraction + keyword matching).

### Docker Compose services (`avoriq/backend/docker-compose.yml`)
| Service  | Image                          | Port (host → container) |
|----------|--------------------------------|--------------------------|
| postgres | `pgvector/pgvector:pg16`       | 5432 → 5432              |
| ollama   | `ollama/ollama:latest`         | 11435 → 11434            |
| backend  | built from local `Dockerfile`  | 8000 → 8000              |

Network: `avoriq-net` (bridge).

## 4. Data shapes (TypeScript + JSON)

### `Scholarship` (frontend)
```
id, name, provider, amount, amountFormatted, deadline,
eligibility: { educationLevel[], gender, familyIncomeMax, states[], castes[], fieldsOfStudy[] },
category: "Government" | "Private" | "NGO" | "International",
description, documents[], officialLink, status, coverage, selectionProcess, benefits,
faqs: [{ question, answer }]
```

### `StudentProfile`
```
name, educationLevel, gender, familyIncome, familyIncomeCategory,
state, disabilityStatus, minorityStatus, caste, fieldOfStudy, interestCategory
```

### Match scoring (`utils/matcher.ts`)
- Education level, gender, family income (vs cap), state domicile, caste, field of study.
- Category boost when `scholarship.category === profile.interestCategory`.
- Filter layer: search query, min amount, category, provider type, sort (amount / deadline / score / id).

## 5. Quick run reference

### Frontend only (current V1)
```bash
cd avoriq/frontend
npm install
npm run dev
# http://localhost:3000
```

### Full stack (frontend + backend + postgres + ollama)
```powershell
# From repo root
powershell -ExecutionPolicy Bypass -File scripts/setup.ps1
```
After setup:
- API: `http://localhost:8000`
- API docs (Swagger): `http://localhost:8000/docs`
- Health: `http://localhost:8000/api/health`
- Ollama: `http://localhost:11435`
- Frontend (separate shell):
  ```bash
  cd avoriq/frontend
  npm run dev
  ```

### Run scraper standalone
```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r avoriq\scripts\requirements.txt
python avoriq\scripts\scrape_scholarships.py --output-dir datasets\scholarships
```

## 7. Notable conventions

- Path alias `@/*` → `avoriq/frontend/*` (see `tsconfig.json`).
- `eslint-config-next` for the frontend; `pyright` config hints at backend `.venv`.
- `.gitignore` keeps `node_modules`, `.next`, Ollama model blobs, Docker volumes, backups, and `avoriq.db` out of git.
- Mock preview banners warn when Firebase or backend is offline.
- Guest users have a **3-message chat limit** (`hooks/useChatLimit.ts`).
- Anonymous users see only the **first 3 scholarships** on `/scholarships`; the rest are blurred behind a sign-in overlay.
- Progress milestones live in `localStorage` under `avoriq_progress` and trigger `canvas-confetti` on the "Funds" stage.
